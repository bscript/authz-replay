Authorized testing only

AuthZ Replayer replays your requests as other identities to find broken access control. Replaying write requests can create, modify, delete, email, or charge - for real.

Only use this against systems you own or are explicitly authorized to test (your own accounts, a bug-bounty program in scope, a staging environment).

authz//replay

Owner markers (A's email / id / private values, one per line). If any shows up in B or unauth, it is flagged BYPASSED.
CSRF header names auto-swapped to B's token on replay (one per line, blank to disable)
Volatile fields stripped before diffing (regex, one per line)

Scope is empty by default - capturing a session auto-adds its origin. Only in-scope requests are intercepted, replayed, and shown.
